## Description

Replace the Manual Trigger node in Lane 1 with a Webhook Trigger node. This makes the n8n workflow listen for incoming HTTP POST requests from the IMAP poller (Task 1) instead of requiring manual activation. The production webhook URL generated by n8n after activation becomes the N8N_WEBHOOK_URL for the poller.

## Acceptance Criteria

- [ ] Updated workflow JSON generated
- [ ] Webhook node configured correctly for POST /email-inbound
- [ ] JSON imports cleanly into n8n without errors
- [ ] After human activates workflow, production webhook URL is available

## Webhook Node Spec

```json
{
  "name": "Webhook Trigger",
  "type": "n8n-nodes-base.webhook",
  "typeVersion": 2,
  "parameters": {
    "httpMethod": "POST",
    "path": "email-inbound",
    "responseMode": "responseNode",
    "options": {}
  }
}
```

**Expected production URL format after activation:**
`https://siteware.app.n8n.cloud/webhook/email-inbound`

**Expected test URL format:**
`https://siteware.app.n8n.cloud/webhook-test/email-inbound`
